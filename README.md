# TOGAF – Fase A: Visión de Arquitectura
Transformación de CentralBank hacia un Neobanco Híbrido  
**Realizado por: Vanesa Acevedo, Laura Chica y Marcela Gómez**

---

## Descripción del Repositorio

Este repositorio contiene toda la documentación correspondiente a la **Fase A – Visión de la Arquitectura** del marco TOGAF, aplicada al ejercicio de transformación de CentralBank hacia un **Neobanco Híbrido**.  
Aquí se centraliza el análisis del problema, objetivos, actores, procesos clave, requisitos, principios y la arquitectura objetivo definida durante esta fase.

---

## Objetivos del Repositorio

- Documentar la Fase A completa según el ADM de TOGAF.  
- Definir el propósito, alcance y visión futura de la arquitectura.  
- Registrar stakeholders, sus preocupaciones y escenarios críticos.  
- Identificar los requisitos de negocio, funcionales, técnicos, datos y calidad.  
- Presentar el modelo de arquitectura resultante, restricciones y principios.  
- Consolidar la matriz de requisitos mapeados a la arquitectura.  
- Preparar la base conceptual para las fases siguientes del ADM.

## ¿Qué es la Fase A de TOGAF? (Descripción corta)

La **Fase A** define la **Visión de Arquitectura**, es decir:

- Qué problema se busca resolver.  
- Qué capacidades y alcance tendrá la arquitectura.  
- Qué interesados participan y qué les preocupa.  
- Qué restricciones existen.  
- Cuál será el estado futuro deseado.  
- Qué requisitos deben ser considerados.  
- Cómo se comunicará la visión (elevator pitch).  

Esta fase marca la dirección estratégica que guiará las Fases B, C y D.
